# CS 255 – System Analysis and Design (DriverPass Project)

## Project Overview
The DriverPass project focuses on designing a system that helps students prepare for their driving tests. The client wanted a solution that combines online practice exams with the ability to schedule in-person driving lessons. The system supports multiple users, including students, instructors, staff, and administrators, each with specific roles.


## Project Artifacts

### Business Requirements Document
This document outlines the system’s purpose, background, goals, and both functional and nonfunctional requirements. It demonstrates the ability to gather and analyze user needs.

### System Design Document
This document includes UML diagrams such as use case, activity, sequence, and class diagrams. It also defines the system’s technical requirements and structure, showing how the system will function and operate.


## Reflection

### What I Did Well
I effectively identified both functional and nonfunctional requirements and organized them clearly. I also created UML diagrams that accurately represent how the system works and how users interact with it. Additionally, I improved my ability to explain technical concepts in a simple and clear way for nontechnical audiences.

### What I Would Improve
If I could revise part of my work, I would improve the technical requirements section by making it more specific and measurable. This would ensure that the system design aligns more closely with real-world implementation standards.

### Interpreting User Needs
I analyzed the needs of different users, such as students, staff, and administrators, and translated those needs into system features and diagrams. Considering user needs is important because it ensures the system is useful, easy to use, and meets its intended purpose.

### Approach to System Design
My approach to designing software is to first understand the problem, then break the system into smaller components. I use UML diagrams and structured planning to visualize the system. In the future, I will continue using these techniques and focus on creating more detailed and testable requirements.
